01 — El problema

¿Qué son las N-Reinas?

Colocar N reinas en un tablero de N×N de modo que ninguna se amenace entre sí. Una reina amenaza a cualquier pieza en su misma fila, columna o diagonal.

Restricción

Sin ataques cruzados

Ningún par de reinas puede compartir fila, columna ni diagonal. Con N=4 hay exactamente 2 soluciones. Con N=8 hay 92 soluciones.

Estrategia

Una reina por fila

Como no puede haber dos reinas en la misma fila, el algoritmo coloca exactamente una reina por fila, recorriendo de fila 0 a fila N-1.

→
Intentar col
for (int col=0; col<n; col++)
?
Verificar
esSeguro(fila, col)
♛
Colocar
tablero[fila] = col
↩
Backtrack
siguiente col del for
02 — El código

Tu método colocarReina()

Java// Punto de entrada — el usuario llama este método
public List<int[]> resolverNReinas(int n) {
    List<int[]> soluciones = new ArrayList<>();
    int[] tablero = new int[n]; // tablero[i] = columna de la reina en fila i
    colocarReina(0, tablero, soluciones, n); // empieza desde fila 0
    return soluciones;
}
Java — Recursión + Backtrackingprivate void colocarReina(int fila, int[] tablero, List<int[]> soluciones, int n) {

    // ── CASO BASE ────────────────────────────────────────────────────────
    if (fila == n) {                    // colocamos todas las reinas
        soluciones.add(tablero.clone());  // guardamos esta solución
        return;                           // regresamos (no hay más que hacer)
    }

    // ── PASO RECURSIVO: probar cada columna en la fila actual ─────────────
    for (int col = 0; col < n; col++) {

        if (esSeguro(fila, col, tablero)) {  // ¿podemos poner la reina aquí?
            tablero[fila] = col;               // COLOCAMOS la reina
            colocarReina(fila + 1, tablero, soluciones, n);  // ← LLAMADA RECURSIVA
        }  // si no funcionó, el for prueba la siguiente columna (backtrack implícito)
    }
    // Al terminar el for sin éxito → esta función regresa, deshaciendo la decisión
}
Java — Función de podaprivate boolean esSeguro(int fila, int col, int[] tablero) {
    for (int i = 0; i < fila; i++) {
        int colColocada = tablero[i];

        if (colColocada == col)  // misma columna
            return false;

        if (Math.abs(colColocada - col) == Math.abs(i - fila)) // diagonal
            return false;
    }
    return true; // ninguna reina anterior ataca → es segura
}
Caso base

fila == n

Cuando la fila es igual a N, todas las N reinas ya están colocadas sin conflicto. Se guarda una copia del tablero y se regresa para seguir buscando más soluciones.

Backtracking

El for como retroceso

Si colocarReina(fila+1) no encontró solución, el for simplemente prueba la siguiente columna — sobrescribiendo tablero[fila]. El backtrack es implícito.

!

Clave: No hay un paso explícito de "deshacer" porque tablero[fila] = col simplemente se sobrescribe en la siguiente iteración del for. El backtracking ocurre cuando la llamada recursiva regresa y el bucle avanza a la siguiente columna.

03 — La poda

Cómo funciona esSeguro()

Esta función revisa todas las reinas ya colocadas (filas 0..fila-1) y verifica si alguna ataca la posición candidata:

Conflicto 1: misma columna

colColocada == col

Si alguna reina anterior está en la misma columna, hay conflicto vertical directo. Dos reinas en la misma columna siempre se atacan.

Conflicto 2: diagonales

|col₁-col₂| == |fila₁-fila₂|

Si la diferencia de columnas es igual a la diferencia de filas, las dos reinas están en la misma diagonal (ascendente o descendente). Un solo abs() cubre ambas diagonales.

✓

No revisa filas porque el algoritmo ya garantiza que cada fila tiene exactamente una reina (una reina por llamada recursiva). La verificación de fila sería redundante.

Tiempo: O(n!)

Poda salva el día

Sin esSeguro() exploraríamos nⁿ nodos. La poda descarta ramas completas antes de explorarlas, reduciendo el trabajo de forma dramática.

Espacio: O(n)

Tablero + pila de llamadas

Solo se guarda un array de tamaño N para el tablero. La pila recursiva tiene profundidad máxima N (una llamada por fila). Muy eficiente en memoria.

07 — Resumen

Los 4 elementos del backtracking

1 — Caso base

if (fila == n) — Cuando hemos colocado una reina en cada fila sin conflictos, tenemos una solución completa. Se guarda y se regresa para seguir buscando más.

2 — Elección

El for (col=0; col<n; col++) representa todas las elecciones posibles para la fila actual. El algoritmo las prueba en orden.

3 — Poda

if (esSeguro(fila, col, tablero)) — Antes de explorar una rama, verificamos si tiene sentido. Si no es seguro, saltamos esa columna sin llamar a la recursión.

4 — Backtrack implícito

No hay un "deshacer" explícito. Cuando colocarReina(fila+1) regresa, el for avanza a la siguiente columna y sobrescribe tablero[fila] — ese es el retroceso.
